## 📋 安全承诺

AI Program Hub 致力于提供安全的程序分享环境。但由于程序由 AI 生成，使用者需要注意以下安全事项。

---

## ⚠️ 重要安全提示

### 对于使用者

1. **审查代码**
   - 运行任何程序前，请先阅读源代码
   - 注意文件操作、网络请求、系统命令等敏感操作
   - 如有疑问，在 Issue 中提问

2. **沙箱测试**
   - 首次运行建议在虚拟环境或沙箱中
   - 不要在生产环境中直接使用
   - 处理敏感数据前先备份

3. **依赖检查**
   - 审查 requirements.txt 中的依赖
   - 使用 `pip-audit` 或 `safety` 扫描已知漏洞
   - 尽量使用标准库

4. **权限控制**
   - 不要用管理员权限运行（除非必要）
   - 限制程序访问的文件目录
   - 注意程序的网络访问

### 对于贡献者

1. **代码规范**
   - 禁止硬编码密钥、密码、Token
   - 禁止未经用户同意的网络请求
   - 禁止访问系统敏感文件
   - 禁止执行系统命令（除非必要且安全）

2. **依赖管理**
   - 只列出必需的依赖
   - 指定确切版本号
   - 避免使用有安全问题的包

3. **数据安全**
   - 不收集用户数据（除非明确说明）
   - 不上传用户文件到外部
   - 处理本地文件要有权限检查

4. **诚实声明**
   - 必须声明程序由 AI 生成
   - 说明已知的安全风险
   - 不提供虚假功能描述

## 🛡️ 安全最佳实践

### 代码示例：安全的文件读取

```python
# ✅ 好的做法
import os

def safe_read_file(filename):
    # 验证文件路径
    if not os.path.exists(filename):
        raise FileNotFoundError("文件不存在")
    
    # 检查文件是否在安全目录
    real_path = os.path.realpath(filename)
    if not real_path.startswith(os.getcwd()):
        raise PermissionError("禁止访问外部文件")
    
    # 安全读取
    with open(filename, 'r', encoding='utf-8') as f:
        return f.read()

# ❌ 不好的做法
def unsafe_read(filename):
    # 没有验证，可能被利用
    with open(filename, 'r') as f:
        return f.read()
```

### 代码示例：安全的输入处理

```python
# ✅ 好的做法
def process_user_input(user_input):
    # 输入验证
    if not user_input or len(user_input) > 1000:
        raise ValueError("输入无效")
    
    # 移除危险字符
    safe_input = user_input.strip()
    
    # 处理
    return safe_input.upper()

# ❌ 不好的做法
def process_input(user_input):
    # 没有验证，直接使用
    return user_input.upper()
```
